Hetzner Server installieren

Bestellung von Servern Link zu Überschrift

  1. IPv4 Option. Nach der Installation wird diese direkt gekündigt. Die IPv4 Adresse bekommt der Server später über den Vswitch .
  2. Server werden immer mit Rescue System bestellt. Wir installieren unsere Maschinen selbst.
  3. Bei der Bestellung unbedingt einen SSH Key hinterlegen .
  4. Status prüfen und Puffer von etwa acht Stunden einplanen. (Lieber einen Tag zu früh bezahlen als ohne Server sein)

Nach der Bestellung Link zu Überschrift

  1. Alle notwendigen Informationen werden uns Email zugesandt.
  2. Prüfe den Fingerprint des Public Keys in der Email. Sollte der Key nicht stimmen, starte den Server erneut und mit korrekten Schlüssel in das Rescue System .
  3. Prüfe beim Einloggen den Host-Key Fingerprint. Sollte dieser nicht korrekt sein, wende dich an den Hetzner Support
  4. Hetzner überprüft seine Hardware vor der Auslieferung. Sollte etwas auffällig sein, Server reklamieren. So der Server bereits eine längere Laufzeit hinter sich hat, die Lebensdauer der SSD überprüfen und gegebenenfalls tauschen lassen.

Installation Link zu Überschrift

  1. Git auf dem Server installieren
  2. Repository holen
  3. Daten beschaffen:

a. FQDN = überlege einen passenden Namen für die Maschine.
a. CRYPT_PASS = lass dir ein sicheres Passwort in deinem Passwortgenerator
generieren.
a. IPV4 = Die Adresse aus der Auslieferungsmail von Hetzner

  1. ./install.sh FQDN CRYPT_PASS IPV4

  2. Neustart, wenn keine Fehlermeldungen kamen.

  3. TODO: STORAGE beschreiben

  4. Remote unlock einrichten.
    Per SSH funktioniert das entsperren der Luks Partitionen bereits. Damit dies automatisch funktioniert, müssen noch alle Tang Server hinzugefügt werden, welche die Partitionen entsperren dürfen. Dazu als erstes die neue IP in der Firewall freischalten und dann folgende Kommandos auf der Kommandozeile eingeben:

1  clevis luks bind -d /dev/md1 tang '{"url": "http://[2003:a:b16:4700:56b2:3ff:fefd:3ec9]"}'
2  # cl07
3  clevis luks bind -d /dev/md1 tang '{"url": "http://[2a01:4f9:5a:5393::2]"}'
4  # cl06
5  clevis luks bind -d /dev/md1 tang '{"url": "http://[2a01:4f8:151:3184::2]"}'
6  # cl05
7  clevis luks bind -d /dev/md1 tang '{"url": "http://[2a01:4f8:151:317c::2]"}'
